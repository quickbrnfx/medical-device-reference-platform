# User Needs

## Overview

User Needs describe the goals that stakeholders expect a system to accomplish.

They represent solution-independent engineering objectives derived from stakeholder analysis, operational context, and stakeholder personas. User Needs define *what stakeholders need to achieve*, not how the system will achieve it.

User Needs establish the bridge between understanding the problem and defining the solution.

---

## Purpose

The purpose of User Needs is to capture stakeholder goals in a manner that is independent of implementation while providing a clear foundation for system requirements.

Well-defined User Needs improve engineering traceability, reduce ambiguity during system development, and help ensure that requirements remain aligned with stakeholder expectations.

---

## When They Are Defined

User Needs are developed during Product Definition after Stakeholder Analysis, Operational Context, and Stakeholder Personas have been completed.

By this stage, the engineering team understands who the stakeholders are, the environment in which the system operates, and the goals those stakeholders are trying to accomplish.

---

## Developing User Needs

Developing User Needs typically involves:

1. Reviewing the Stakeholder Analysis.
2. Understanding the Operational Context.
3. Identifying the goals of each primary stakeholder persona.
4. Expressing those goals as solution-independent statements.
5. Reviewing the needs for clarity, completeness, and traceability.

Each User Need should represent a stakeholder objective rather than a feature, task, or design decision.

---

## Characteristics of Effective User Needs

Effective User Needs should be:

- Focused on stakeholder goals.
- Independent of implementation.
- Clear and unambiguous.
- Stable throughout product development.
- Traceable to one or more System Requirements.

User Needs should describe **what stakeholders need to accomplish**, not **how the system will accomplish it**.

---

## Relationship to Other Artifacts

User Needs complete Product Definition and provide the primary input to Requirements Engineering.

```text
Stakeholder Analysis
        │
        ▼
Operational Context
        │
        ▼
Stakeholder Personas
        │
        ▼
User Needs
        │
        ▼
System Requirements
        │
        ▼
Architecture
```

Stakeholder Analysis identifies who has an interest in the system.

Operational Context defines where and how the system operates.

Stakeholder Personas identify the primary operational stakeholders.

User Needs describe the goals those stakeholders are trying to achieve.

System Requirements define the capabilities the system must provide to satisfy those goals.
