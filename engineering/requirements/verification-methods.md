# Verification Methods

## Overview

Verification determines whether a system satisfies its specified requirements through objective evidence.

Each System Requirement should have one or more defined verification methods that demonstrate the requirement has been successfully implemented.

Verification answers the question:

> Did we build the system correctly?

---

## Purpose

The purpose of defining Verification Methods is to establish a consistent approach for demonstrating compliance with System Requirements.

Selecting an appropriate verification method early in development improves requirement quality, guides test planning, and reduces ambiguity during system verification.

---

## Verification Principles

Verification should be:

- Objective
- Repeatable
- Traceable
- Appropriate for the requirement being evaluated

Verification demonstrates compliance with requirements. It does not determine whether the correct requirements were defined.

---

## Verification Methods

Four verification methods are commonly used throughout systems engineering.

| Method | Description |
|----------|-------------|
| Inspection | Verification by visual examination of documentation, software, hardware, or other engineering artifacts. |
| Analysis | Verification through calculation, modeling, simulation, or engineering evaluation. |
| Demonstration | Verification by observing the system performing an expected function without detailed measurement. |
| Test | Verification through controlled execution with defined procedures, inputs, and acceptance criteria. |

A requirement may be verified using one or more methods depending on its nature.

---

## Selecting Verification Methods

The selected verification method should provide sufficient objective evidence that the requirement has been satisfied.

Selection depends on factors such as:

- Requirement type
- System complexity
- Associated risk
- Available evidence
- Practicality

Verification methods should be identified as requirements are developed rather than after implementation.

---

## Relationship to Other Artifacts

Verification Methods support the transition from System Requirements to Verification Planning.

```text
User Needs
      │
      ▼
System Requirements
      │
      ▼
Verification Method
      │
      ▼
Verification Plan
      │
      ▼
Verification Results
```

Every System Requirement should have an identified verification approach that ultimately produces objective evidence of compliance.
