# Templates

## Overview

The Templates section provides practical starting points for creating the engineering artifacts described throughout this repository.

Where the Engineering sections explain *why* an artifact exists and *what engineering question it answers*, the Templates demonstrate *how to create one*.

Each template is intended to accelerate engineering work while maintaining consistency, traceability, and engineering rigor.

Templates should be adapted to the needs of individual projects rather than completed mechanically.

---

## Purpose

The purpose of this section is to help engineering teams translate engineering methodology into engineering practice.

Each template provides:

- The engineering question the artifact answers.
- Guidance for completing each section.
- A reusable document structure.
- A worked reference example.

---

## How to Use the Templates

Templates are intended to be used after understanding the associated engineering methodology.

The recommended workflow is:

1. Read the engineering guidance.
2. Understand the engineering objective.
3. Select the appropriate template.
4. Adapt the template to the project.
5. Review the completed artifact for clarity, completeness, and traceability.

Templates should support engineering thinking rather than replace it.

---

## Template Organization

Templates generally follow the same structure:

1. Engineering Question
2. Purpose
3. Instructions
4. Template
5. Reference Example
6. Common Pitfalls

This consistent format helps reinforce the engineering methodology presented throughout the repository.

---

## Relationship to the Engineering Framework

Templates correspond directly to the engineering disciplines described throughout the repository.

```text
Engineering Methodology
          │
          ▼
Engineering Artifact
          │
          ▼
Template
          │
          ▼
Completed Project Artifact
```

The methodology explains *why* an artifact exists.

The template helps engineers create it.
