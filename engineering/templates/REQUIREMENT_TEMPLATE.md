# Architecture Decision Record (ADR) Template

---

# ADR Identifier

| Attribute | Value |
|----------|-------|
| **ID** | ADR-XXX |
| **Title** | Concise decision title |
| **Status** | Proposed / Accepted / Superseded / Deprecated |
| **Date** | YYYY-MM-DD |
| **Decision Makers** | Engineering team, project maintainer, etc. |

---

# Decision Statement

Provide a one- or two-sentence statement describing the engineering decision.

Example:

> The project adopts a layered software architecture for the Version 1 reference platform.

---

# Context

Describe the engineering problem or question that required a decision.

Include relevant background, constraints, assumptions, and references to supporting engineering artifacts.

If a Decision Analysis exists, reference it here rather than duplicating its contents.

---

# Decision

Describe the accepted engineering decision.

Focus on **what was decided**, not how the decision was reached.

---

# Decision Rationale

Explain why this decision was selected.

Summarize the primary engineering considerations without reproducing detailed trade studies.

---

# Consequences

## Positive

Document the expected engineering benefits.

Examples:

- Simplifies verification
- Reduces system complexity
- Improves maintainability
- Supports future expansion

## Negative

Document known trade-offs or limitations introduced by the decision.

---

# Alternatives

List any significant alternatives considered.

If detailed evaluation exists elsewhere, reference the corresponding Decision Analysis.

---

# Affected Artifacts

Identify engineering artifacts that are established or modified by this decision.

Examples:

- System Requirements
- Architecture
- Verification Strategy
- Cybersecurity Architecture
- Risk Analysis

---

# Traceability

## Inputs

Reference engineering artifacts that informed the decision.

Examples:

- Project Charter
- Engineering Lifecycle
- Decision Analysis
- System Requirements

---

## Outputs

Reference engineering artifacts that depend upon this decision.

Examples:

- Architecture
- Verification
- Risk Engineering
- Implementation

---

# References

List supporting documents, standards, and engineering artifacts.
