# Connected Physiological Monitor

## Overview

The Connected Physiological Monitor is the first complete reference platform within the Medical Device Engineering Reference repository.

Its purpose is to demonstrate how the engineering methodology described throughout this repository can be applied to the development of a realistic connected medical device.

Rather than serving as a production-ready design, this platform illustrates the engineering decisions, artifacts, traceability, and documentation that accompany modern medical device development.

---

## Purpose

This reference platform demonstrates how engineering activities progress from product definition through verification while maintaining traceability between engineering artifacts.

It provides practical examples of:

- Product Definition
- Requirements Engineering
- System Architecture
- Human Factors Engineering
- Cybersecurity Engineering
- Verification Engineering
- Hardware Development
- Firmware Development
- Software Development

---

## Product Vision

Develop a wearable physiological monitoring system capable of acquiring physiological signals, processing those signals locally, securely transmitting relevant information to external systems, and supporting continuous patient monitoring in representative healthcare environments.

The platform emphasizes engineering methodology rather than product performance.

---

## System Scope

The Connected Physiological Monitor includes representative examples of:

- Physiological sensing
- Embedded firmware
- Device hardware
- Wireless communication
- Mobile or gateway connectivity
- Cloud integration
- Clinical monitoring workflows
- Cybersecurity considerations
- Human Factors engineering
- Verification activities

The platform intentionally simplifies many implementation details in order to focus on engineering decisions and traceability.
